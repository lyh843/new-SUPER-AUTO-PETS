# Super Auto Pets

## 一、游戏基本要素：

### 1. 游戏主页面：

<img src=".\pictures\image-20251018213125529.png" alt="image-20251018213125529" style="zoom: 15%;" />

#### 中间：

1. **游玩：** 开始新一轮的对决
2. **宠物：** 打开宠物图鉴
3. **战绩：** 查询过往游玩记录得分

### 2. 宠物图鉴：

<img src=".\pictures\image-20251018213310056.png" alt="image-20251018213310056" style="zoom:15%;" />

#### 要素：

- 宠物 + 食物的分级
- 点击对应宠物时，屏幕下方出现宠物介绍。
  - 宠物的攻击力
  - 宠物的声明
  - 宠物在不同等级下的技能
- 点击食物时，出现食物介绍

### 3. 备战区：

<img src=".\pictures\image-20251018213422623.png" alt="image-20251018213422623" style="zoom:15%;" />

#### 最上面的属性栏：

- 当前回合的金币数（用于购买宠物和食物）

- 血量
- 当前的回合数
- 获胜的局数

#### 偏上的那一栏：

- 部署点位（可以给一个临时点位，方便满动物时的顺序调换）

#### 偏下的一栏：

- 商店，三个宠物可供购买（要实现查看属性和购买两个不同操作）
- 右侧是道具（同样实现两个操作）

#### 最下的一栏：

- 刷新商店（宠物+道具）
- 冻结：将该动物保留至下一回合购买
- 结束回合：退出商店，进行作战

### 4. 战场：

<img src=".\pictures\image-20251018213510185.png" alt="image-20251018213510185" style="zoom:15%;" />

#### 最上的一栏

- 游玩：双方轮流进行一回合的行动
- 自动播放：双方轮流行动，直至分出输赢

#### 中间一栏

- 战斗区域：双方宠物进行对战


#### 最下的一栏

- 左边：我方状态栏（回合数+获胜局数+剩余血量）
- 右边：敌方（人机 / 联机对手）状态栏

## 二、代码主要实现

### 1. GUI-Qt 3人

- 角色图以及背景图
- 宠物打斗的动画脚本
- 简单的特效效果

### 2. 角色类 2人

- 一个 `animals`的基类
  - 基本属性：生命、攻击力、买入、卖出价格、技能种类、星级、等级（瞬发、条件触发、被动）
  - 基本函数：技能实现、返回自身基本属性
- 一个`food`的基类

### 3. 商店与战场逻辑 2人

- 商店
  - 购买、冻结、出售
  - 角色的升级
- 战场
  - 食物效果的触发
  - 瞬发技能的使用

## 二、模块职责说明

| 模块    | 职责                   | 说明                                             |
| ------- | ---------------------- | ------------------------------------------------ |
| app    | 应用控制与窗口生命周期 | 负责创建 QApplication、加载资源、管理场景切换    |
| model  | 游戏数据与逻辑对象     | 不依赖 Qt，对应游戏的抽象数据层                  |
| engine | 战斗与事件调度         | 管理宠物行动顺序、触发效果、计算战斗结果         |
| ui     | 图形界面层             | 使用 Qt Widgets 或 QML 构建界面，展示 model 状态 |
